# ===== STEP 2: Test before deployment =====

# In another terminal, test health

# ===== TROUBLESHOOTING =====

# If build fails, check:
# 1. Logs in Render dashboard (Logs tab)
# 2. requirements.txt syntax
# 3. Python version compatibility

# If API doesn't respond:
# 1. Check service is "Active" in Render dashboard
# 2. Wait 30-60 seconds for cold start
# 3. Check OPENWEATHER_API_KEY is set
# 4. View logs for errors

# To view Render logs (use this after getting .onrender.com URL):
curl -s https://agroai-api.onrender.com/healthz

# To manually rebuild on Render:
# Dashboard → agroai-api → Manual Deploy → Deploy latest commit

# ===== MAKING UPDATES =====

# Every time you push to main, Render automatically redeploys!
git add .
git commit -m "Your message"
git push origin main

# Render will:
# 1. Detect the push
# 2. Build the application
# 3. Run tests
# 4. Deploy the new version (2-5 minutes)
